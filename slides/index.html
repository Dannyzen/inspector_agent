Inspector Agent

A multi-agent system for network security

The Problem

How can we proactively monitor a system for malicious network activity?

The Solution

A multi-agent system built with the Google Agent Development Kit (ADK) that can:

  • Inspect network connections
  • Analyze processes for threats
  • Provide interactive remediation

Agentic Architecture

The Inspector Agent is composed of multiple specialized agents that work together:

  • Inspector: Monitors network connections.
  • Threat Analyzer: Assesses the risk of each process using a rules-based engine.
  • Remediation Tool: Kills processes that are deemed high-risk.

How it Works

  1. The `Inspector` agent scans for listening processes.
  2. The `ThreatAnalyzer` agent assesses the risk of each process using a set of predefined rules.
  3. If a threat is detected, the `Remediation` tool is used to kill the process.

Novel ADK Implementation

The Inspector Agent demonstrates several key features of the ADK:

  • Multi-Agent Systems: The agent is composed of multiple specialized agents that collaborate to solve a complex problem.
  • Stateful Agents: The agent uses session state to track processes and maintain a history of network activity.
  • Tool Use: The agent uses a custom tool to interact with the system and remediate threats.

Multi-Agent Collaboration

The `InspectorAgent` is a `SequentialAgent` that orchestrates the workflow between the `Inspector` and `ThreatAnalyzer` agents. This demonstrates how the ADK can be used to build complex systems from smaller, reusable components.

The `SequentialAgent` ensures that the agents are executed in a specific order, which is crucial for the proper functioning of the `inspector_agent`. The `Inspector` must run first to gather the process information, which is then passed to the `ThreatAnalyzer` for analysis.

Stateful Process Tracking

The `Inspector` agent uses the `context.session.state` to keep track of the processes it has already seen. This allows it to only report on new processes, which is a more efficient way to monitor for threats.

Dynamic Remediation

The `ThreatAnalyzer` agent uses the `remediation_tool` to kill processes that are deemed high-risk. This demonstrates how the ADK can be used to build agents that can not only reason about problems, but also take action to solve them.

Agent Types

  • `BaseAgent`: The `Inspector` is a `BaseAgent`, which is the simplest agent type. It's used for simple, single-purpose tasks like scanning for processes.
  • Rules-Based Agent: The `ThreatAnalyzer` is a standard Python class that uses a dictionary of rules to identify threats. This is a simple and effective approach for known threats.
  • `SequentialAgent`: The `InspectorAgent` is a `SequentialAgent`, which is used to orchestrate a series of sub-agents. This allows us to create a complex workflow from smaller, reusable components.

How to Use It

  1. Start the agent: ./.venv/bin/python3 main.py
  2. Simulate a threat: nc -l 8083
  3. Observe the output in the agent's terminal as the threat is detected and remediated.

Example Scenarios

  • Detecting and killing a malicious process that has opened a port.
  • Identifying and whitelisting a legitimate process that is not yet known to the system.
  • Monitoring a server for unauthorized services.
